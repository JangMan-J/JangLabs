Arch Linux Install Guide

Laptop · RTX 4090 · Steam-dependent project · btrfs + snapshots
Phase 1 Bootable USB (Ventoy)
yay -S ventoy-bin

Or download from ventoy.net, extract, and run from the extracted directory.

Identify the USB — verify this is the USB, not your disk:

lsblk

Install Ventoy (this nukes the USB):

sudo Ventoy2Disk.sh -i /dev/sdX

After install the USB has two partitions: a tiny VTOYEFI boot partition and a large Ventoy data partition (exFAT). The data partition auto-mounts. Drop archlinux-x86_64.iso onto it as a regular file — no dd, no formatting tricks.

Before booting Ventoy: Disable Secure Boot in firmware. Also required for Arch generally.

For future Ventoy updates (preserves data — never use -i again):

sudo Ventoy2Disk.sh -u /dev/sdX
Phase 2 Boot Arch ISO & Install

Reboot. Boot from USB. Pick the Arch ISO from the Ventoy menu.

In the Arch live environment, confirm device identity again — names can shuffle between boots:

lsblk
Most likely way to lose the backup: wiping the USB by accident. Double-check device names.

Connect to network:

# WiFi (if needed) iwctl # Verify connectivity ping archlinux.org

Run archinstall with these selections:

SettingValue
Disk configBest-effort default → wipe entire internal disk (NOT the USB) → btrfs with snapshot-friendly subvolume layout (@, @home, @log, @pkg). archinstall doesn't create a snapshots subvolume — we add @snapshots manually in Phase 3.
BootloaderGRUB (not systemd-boot — GRUB is required for snapshot rollback from boot menu)
ProfileKDE or GNOME, minimal package selection
AudioPipeWire
NetworkNetworkManager
Graphics driverNvidia (open kernel module) — installs nvidia-open-dkms + utils, configures nvidia-drm.modeset=1
Additional packagesLeave empty (added post-install)
Additional reposEnable multilib (essential for native Steam + 32-bit deps)
AccountsSet hostname, root password, your user account

Let it finish, reboot, remove the USB.

Hybrid graphics note: leave BIOS on discrete-only. The live ISO boots fine on nouveau via UEFI framebuffer; once archinstall configures nvidia-open-dkms, your first installed-OS boot is already on the proprietary driver.

Phase 3 First Boot — Foundational Setup

Sync mirrors, update, verify multilib, install build tooling:

sudo pacman -Syu # Verify multilib is enabled — should see [multilib] section grep -A1 multilib /etc/pacman.conf # Base build tooling (required for AUR helper + DKMS rebuilds) sudo pacman -S --needed base-devel git
AUR Helper (yay)
git clone https://aur.archlinux.org/yay.git cd yay makepkg -si cd ~ && rm -rf yay
Create @snapshots Subvolume

archinstall doesn't create a snapshots subvolume. Create it now as a top-level peer of @ and @home so snapshots stay independent of root — if you ever wipe or replace @, snapshots survive.

# Find your btrfs partition lsblk -f # Mount the top-level btrfs volume (subvolid=5) sudo mount -o subvolid=5 /dev/nvme0n1pX /mnt # Create @snapshots alongside the other top-level subvolumes sudo btrfs subvolume create /mnt/@snapshots # Verify it's there next to @, @home, @log, @pkg sudo btrfs subvolume list /mnt sudo umount /mnt

Add it to /etc/fstab. Copy the options from an existing subvolume line (like @home) and change the subvol name and mountpoint:

sudo mkdir -p /.snapshots

Add this line to /etc/fstab (match UUID and options from your other btrfs entries):

UUID=<same-as-other-btrfs-entries> /.snapshots btrfs subvol=@snapshots,compress=zstd:1,noatime 0 0
sudo mount -a # Confirm it mounted findmnt /.snapshots
Snapshot / Rollback Stack
sudo pacman -S snapper snap-pac grub-btrfs inotify-tools # Configure snapper for root sudo snapper -c root create-config /

Snapper just created its own .snapshots subvolume nested under @, ignoring the one we set up. Delete it and remount ours:

# Remove snapper's auto-created nested subvolume sudo btrfs subvolume delete /.snapshots # Recreate the mountpoint and remount our top-level @snapshots sudo mkdir /.snapshots sudo mount -a sudo chmod 750 /.snapshots

Edit retention policy:

sudoedit /etc/snapper/configs/root

Recommended values:

TIMELINE_LIMIT_HOURLY="5" TIMELINE_LIMIT_DAILY="7" TIMELINE_LIMIT_WEEKLY="4" TIMELINE_LIMIT_MONTHLY="2" TIMELINE_LIMIT_YEARLY="0" NUMBER_LIMIT="20" NUMBER_LIMIT_IMPORTANT="10"

Enable timers and grub-btrfs:

sudo systemctl enable --now snapper-timeline.timer snapper-cleanup.timer # grub-btrfs watches snapshot dir and regenerates GRUB menu sudo systemctl enable --now grub-btrfsd.service
Test Rollback (do this before trusting it)

Install something disposable, reboot into pre-snapshot from GRUB submenu, roll back, verify:

# This creates a pre/post snapshot pair via snap-pac sudo pacman -S cowsay # Reboot → hold Shift → "Arch Linux snapshots" submenu # Boot the pre-cowsay snapshot, then: sudo snapper rollback # Reboot again, verify it's gone: which cowsay # should return nothing
Pacman Hygiene — AUR Rebuild Detection
sudo pacman -S devtools

Create the pacman hook:

sudo tee /etc/pacman.d/hooks/90-checkrebuild.hook <<'EOF' [Trigger] Operation = Upgrade Type = Package Target = * [Action] Description = Checking for packages needing rebuild... When = PostTransaction Exec = /usr/bin/checkrebuild EOF

Now every pacman -Syu ends with a list of packages whose libs got yanked. Rebuild them with yay -S <pkg> --rebuild or yay -Sua --rebuild for all AUR updates.

Subscribe to arch-announce mailing list — low traffic, only manual-intervention notices.
Phase 4 NVIDIA Verification & Tuning

archinstall set up the basics. Verify and add what's missing:

# Confirm what's already installed pacman -Qs nvidia # Add anything missing for full perf sudo pacman -S --needed \ nvidia-open-dkms nvidia-utils lib32-nvidia-utils nvidia-settings \ libva-nvidia-driver \ vulkan-icd-loader lib32-vulkan-icd-loader \ egl-wayland

Verify kernel parameters:

cat /proc/cmdline | grep -o 'nvidia-drm[^ ]*' # Should show: nvidia-drm.modeset=1 nvidia-drm.fbdev=1

If fbdev is missing, edit /etc/default/grub and add it to GRUB_CMDLINE_LINUX_DEFAULT, then:

sudo grub-mkconfig -o /boot/grub/grub.cfg

Verify mkinitcpio modules:

grep ^MODULES /etc/mkinitcpio.conf # Should be: MODULES=(nvidia nvidia_modeset nvidia_uvm nvidia_drm)

Verify kms hook is NOT in the HOOKS line (would conflict):

grep ^HOOKS /etc/mkinitcpio.conf # If 'kms' is there, remove it

If you edited either file:

sudo mkinitcpio -P

Enable power management services:

sudo systemctl enable nvidia-powerd.service sudo systemctl enable nvidia-suspend.service nvidia-resume.service nvidia-hibernate.service

Verify everything works:

nvidia-smi # shows driver + RTX 4090 glxinfo | grep "OpenGL renderer" # NVIDIA + RTX 4090 vulkaninfo --summary | grep deviceName # lists the 4090

Reboot if you changed kernel cmdline or mkinitcpio. Confirm Wayland session in KDE/GNOME after reboot.

If you do ML/compute work (optional):

sudo pacman -S cuda cudnn nvidia-container-toolkit
Phase 5 Steam (Project Hard Dependency)

Multilib was enabled in Phase 2 and verified in Phase 3, so 32-bit packages are available.

Install native Steam from official repos (NOT Flatpak — your project requires host-level access to Steam dirs and the 32-bit library stack):

sudo pacman -S steam # When the dependency picker prompts: # lib32-vulkan-radeon → skip (you have NVIDIA) # lib32-nvidia-utils → select (already installed in Phase 4)

32-bit graphics + audio runtime libs the project may need:

sudo pacman -S --needed \ lib32-mesa lib32-vulkan-icd-loader lib32-nvidia-utils \ lib32-libpulse lib32-alsa-plugins lib32-libxcomposite \ lib32-gtk3 lib32-libxinerama lib32-libxrandr \ ttf-liberation

Proton-related extras:

sudo pacman -S --needed wine-staging winetricks yay -S protontricks

Game performance auto-tuner:

sudo pacman -S gamemode lib32-gamemode # Launch via: gamemoderun <command> # Or in Steam launch options: gamemoderun %command%

Launch Steam, log in, verify it runs. Confirm the 32-bit library paths exist:

ls /usr/lib32/ | head ls ~/.steam/steam/ubuntu12_32/
Phase 6 Flatpak for GUI App Churn
sudo pacman -S flatpak flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo

Use Flatpak for GUI apps you want fresh and disposable (browsers, editors, media tools). Use AUR/pacman for CLI/dev tools. Don't install Steam via Flatpak.

Phase 7 Sanity Checklist
  • pacman -Syu runs cleanly
  • nvidia-smi shows the 4090
  • KDE/GNOME on Wayland session, no tearing on fullscreen video
  • Snapshot rollback test passed (Phase 3)
  • checkrebuild runs at end of pacman transactions
  • Steam launches, logs in, downloads a small free game successfully
  • Multilib enabled: grep -A1 '^\[multilib\]' /etc/pacman.conf
  • gamemoderun glxgears shows GPU clocked up via nvidia-smi
  • arch-announce subscription confirmed
